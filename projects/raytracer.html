Ray Tracing - C++

Background

If you're not familiar with ray tracing, there are tons of great research papers out there which will describe the process in great detail, but I will provide a brief overview. The idea is to model an object (or scene) by utilizing the nature of vision, except in reverse. Humans see colors as light bounces off of objects in the the world into our eyes, or a camera. Ray tracing an image is the process of sending rays from the camera into the world to see what they would have hit and create a representation of that information. I am primarily concerned with ray tracing scenes and creating images at this point.

Project Details

I started this project for my Advanced Computer Graphics course at Rose, and will continue improving it as I find time. The ray tracer uses obj data to describe a scene, including model geometry and lights, and supports material files for colors and reflections. Other properties are not yet implemented (such as refraction and glossy reflections), but are left as a possible future extension. Intersections utilize a bounding volume hierarchy, with spatial splitting, to improve the runtime. It will create ppm image files of the input scene.

Source Code

The source is available from GitHub. It was designed (and tested) on an Ubuntu 12.04 system, and includes a makefile. I suspect it could easily be ported to Windows.
